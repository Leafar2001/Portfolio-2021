Developing a fun, interactive and simplistic flying game on the web to kill some time.

Plane Game is a simple browser-based game where a user can use the WASD arrowkeys to fly around, avoiding collisions to gain points. There are multiple plane types to choose from, each of them offering different flying speeds.

The project

This project originated as a challenge I got from a school assignment, where we had to show some form of user input to influence this project. I thought it would be a fun idea to make some form of a game, as I had never made a game before. As user input I used WASD keys as it was the most logical option to me, which was later confirmed by some user tests.

Design

Since this project was meant to be done in a couple of days, I didn't spend too much time designing a prototype and just went with developing the vision I had. Using simple elements to visualize a plane game such as clouds, a sky-colored background, and making the clouds move in the background I was able to create a simple design I was happy with, later on adding some fun music for extra immersion.

I also opted to put some controls in the center of the screen so the user instantly sees what they have to do to play the game, and put a collapsable control panel to the left for extra options like different plane types.

Plane Game - UI

Testing

When I was done developing the core features such as being able to control the plane by using the keyboard, I went and asked some colleague students and friends what they thought. I asked them how they would feel about a collision system and if they were able to help me test this. After a while, I managed to develop a sweet spot for the collision system. Even though it didn't have a AAA-game accuracy system, it got the job done.

Plane Game - collisions

Development

I started off creating the user interface with a subtle background and basic functionality like being able to control a plane with the WASD keys. Later on, I was able to add other planes with different speed limits. Once i had the player controlled planes down, I was able to add a collision system by adding other planes coming from the opposite directions, and comparing the coordinates of these planes to see if they had collided with the player's plane. Points were simply calculated by a timer, where u would also lose points for every collision. Lastly I added some standard background music for extra immersion.

Results

I'm happy with the result I was able to get within the limited time frame of a week. All the functionality like different plane types and collisions were working and the game felt somewhat "complete" for a short project. Unfortunately the collision system isn't insanely accurate, but regardless the users I tested the end product with liked the game.

Web app

Developing this game helped me learn alot about approaching user input with JavaScript, aswell as thinking about user experience and how to make a game immersive by using different elements such as visual effects and audio. It helped me get comfortable with using JavaScript to bind object coordinates and use these values to determine things such as obstacle collision.
